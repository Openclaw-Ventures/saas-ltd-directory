## What Jaisev Needs to Do

### Option A: Netlify CLI Deploy (Recommended, 5 min)

1. Go to https://app.netlify.com → Log in (or sign up free)
2. Go to **User Settings → Applications → Personal access tokens**
3. Click **New access token**, name it `saas-ltd-deploy`, copy the token
4. On the server, run:
   ```bash
   cd ~/clawd-forge/active/saas-ltd-directory
   NETLIFY_AUTH_TOKEN=<paste-token-here> netlify deploy --dir=site/public --prod
   ```
5. When prompted to create a new site, say Yes
6. Save the resulting site URL and site ID

Then set the secrets for GitHub Actions:
```bash
# After getting site ID from step 5:
gh secret set NETLIFY_AUTH_TOKEN -R Jaisev-Sachdev/saas-ltd-directory
gh secret set NETLIFY_SITE_ID -R Jaisev-Sachdev/saas-ltd-directory
```

### Option B: Netlify Drop (No CLI, 2 min)

1. Go to https://app.netlify.com/drop
2. Zip the public folder first:
   ```bash
   cd ~/clawd-forge/active/saas-ltd-directory/site
   zip -r /tmp/saas-ltd-public.zip public/
   ```
3. Drag and drop the `public/` folder (or the zip) onto the Netlify Drop page
4. Site goes live immediately with a random `.netlify.app` URL
5. Optionally: set a custom domain in Netlify site settings

### Option C: Vercel (Alternative, 3 min)

1. Go to https://vercel.com/account/tokens
2. Create a token, name it `saas-ltd-deploy`
3. Run:
   ```bash
   cd ~/clawd-forge/active/saas-ltd-directory
   VERCEL_TOKEN=<paste-token-here> vercel deploy site/public/ --prod --token=<paste-token-here>
   ```

### Fix GitHub Actions Workflow Push

The current PAT lacks `workflow` scope, so `.github/workflows/nightly-rebuild.yml` couldn't be pushed.

**Fix:**
1. Go to https://github.com/settings/tokens
2. Edit the existing PAT → check the `workflow` scope → Save
3. Then run:
   ```bash
   cd ~/clawd-forge/active/saas-ltd-directory
   git add .github/workflows/nightly-rebuild.yml
   git commit -m "Add nightly rebuild workflow"
   git push
   ```

The workflow file is already created locally at `.github/workflows/nightly-rebuild.yml`.
